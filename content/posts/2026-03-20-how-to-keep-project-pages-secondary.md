很多技术主页都默认把项目放在非常靠前的位置，这对求职型站点没有问题，但不一定适合长期写作型主页。

## 项目不该主导首页节奏

项目天然是离散的，它们往往：

- 更新频率低
- 生命周期不一致
- 很难形成稳定的阅读路径

而文章更适合形成时间线和主题线。

## 项目的合适位置

项目适合承担两件事：

1. 为文章提供“实践来源”
2. 为读者提供“继续深挖的仓库入口”

当项目放到辅助位置，首页会更集中，读者也更容易先进入内容本身。

```go
type ProjectRole struct {
    SupportsWriting bool
    ShowsPractice   bool
}
```

这不是为了弱化项目，而是让站点更符合“技术记录主页”的定位。
